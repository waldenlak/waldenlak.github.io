一、创建个人仓库

（一）本地

（二）浏览器GitHub Page

1、非QQ邮箱注册GitHub帐号

2、点击New repository创建新仓库

4、勾选README

5、Set up in Desktop或打开GitHub Desktop（下载应用），Clone到本地，在D:\STUDY\BLOG下生成username.github.io文件夹。

二、搭建Git，连接Github与本地

1、Git安装过程详解 Windows系统

2、将Git与GitHub帐号绑定：初始化/配置/修改user.name 和user.email (Git Bash & Git cmd 的区别)

三、安装Node.js、npm、cnpm、Hexo

（一）1、nodejs下载安装，2、配置环境变量，检测node.js的环境变量是否配置成功，

（四）可能出现的bug以及解决方案

（1）安装npm报错

如果没有梯子，可以替换npm源为淘宝提供的npm镜像源以实现国内加速。 使用Node.js（安装Hexo）时出现了 rollbackFailedOptional 的解决方法——RoyalRay:修改网络代理缓月:npx hexo init blog

一、在D:\BLOG\username.github.io打开Windows PowerShell命令行：

  1. 重新配置npm代理与git代理：
    npm config delete proxy
    npm config delete https-proxy
    git config --global --unset http.proxy
    git config --global --unset https.proxy
  2. 设置proxy、https-proxy为null：
    npm config get proxy
    npm config get https-proxy
    保证两个命令的返回值都为null，继续执行以下两个命令：
    npm config set proxy null
    npm config set https-proxy null
  3. 设置nodejs文件夹里prefix、cache路径：
    npm config set prefix "D:\Program Files\nodejs\node\_global"
    npm config set cache "D:\Program Files\nodejs\node\_cache"
  4. 检查npm的配置:
    npm config ls （缺少了镜像配置electron_mirror = “https://npm.taobao.org/mirrors/electron/”）

二、以管理员身份启动cmd(管理员：Windows PowerShell ISE)

  1. 再次安装cnpm淘宝镜像:
    npm install -g cnpm --registry=https://registry.npm.taobao.org

  2. 检查 D:\Program Files\nodejs\node\_global 确实安装了cnpm

  3. 检查淘宝镜像是否安装成功/检查cnpm版本号:
    cnpm -v

  4. 检查registry = https://registry.npm.taobao.org：
    npm config get registry

  5. 修改npm的资源镜像链接为cnpm：
    npm config set registry https://registry.npm.taobao.org
    npm config get registry

    此后所有npm命令替换为cnpm即可。

三、cd进入网站文件夹根目录

  1. 再次安装Hexo:
    cd BLOG
    cnpm install -g hexo-cli
  2. 执行 Hexo的两种方式：
    （1）npx hexo
    （2）将 Hexo 所在的目录下的 node_modules 添加到环境变量之中即可直接使用 hexo：
    echo 'PATH="$PATH:./node\_modules/.bin"' &gt;&gt; ~/.profile
  3. 输入hexo -v验证是否安装成功。

（2）npm install报错

无法加载文件 C:\Program Files\nodejs\node_global\cnpm.ps1，因为在此系统上禁止运行脚本。

1、在终端执行：get-ExecutionPolicy，显示Restricted（表示状态是禁止的）
2、在终端执行：set-ExecutionPolicy RemoteSigned
3、在终端执行：get-ExecutionPolicy，显示RemoteSigned
4、检查npm配置（.npmrc配置文件：）

（五）搭建HEXO框架

1、进入文件夹：cd HEXO
2、初始化文件夹：hexo init HEXO
3、安装必要依赖组件：cnpm install
4、本地预览是否搭建成功：hexo s

1、切换主题

（1）操作步骤

①安装主题，配置样式。安装butterfly主题

②清除缓存，验证主题：
hexo clean
hexo s --debug 或者 hexo g & hexo s

⑤部署网站，同步仓库：hexo g -d

（2）出错解决

Warning: Accessing non-existent property

警告缺少模块依赖：无伤大雅。

主页空白出现如下字符:

extends includes/layout.pug block content include includes/recent-posts.pug include includes/partial

查看 npm 安装各 hexo 插件的情况：（进入网站根目录，管理员执行）npm ls --depth 0

⚪出现missing说明缺少hexo所需的插件：

解决办法：重新单独安装缺少的插件即可
npm i hexo-deployer-git --save
npm i hexo-browsersync --save
安装后再次npm ls --depth 0检查是否安装完成
无问题后重新生成预览（hexo s）或部署（hexo g）

⚪node_modules目录中已经安装了一个package，但是package.json中并没有对该package做依赖，那么这个package就应该被删除。这时如果执行npm ls命令则指示有一个package not used。
为了清理代码，需要执行npm prune

⚪需要安装 pug 以及 stylus 的渲染器
npm install hexo-renderer-pug hexo-renderer-stylus --save

样式失效，hexo g报错：

报错指出vendor相关问题。
解决方案：在主题配置文件找到vendor相关配置，设为false。
css\_prefix: false
