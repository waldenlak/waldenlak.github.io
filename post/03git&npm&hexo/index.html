本地：工作目录d:\SITE，持有实际文件； 暂存区Index，临时保存改动；本地仓库HEAD，指向最新提交的结果。
远端：默认分支/主分支master；其他分支。

Git（-GitHub）命令

workflow工作流

文件夹管理

新建工作目录（文件夹）：mkdir SITE
进入d:\SITE目录下准备执行命令：cd d:\SITE

仓库管理

克隆远端仓库：git clone git@github.com:***/***.github.io.git
克隆远端仓库到SITE文件夹：git clone https://github.com/***/***.git SITE

分支管理

创建并切换至新分支：git checkout -b 'branch'（'branch'分支名）
切换另一分支：git checkout master
预览分支差异：git diff 'source\_branch' 'target\_branch'
其它分支合并至当前分支：git merge master（需要手动合并冲突conflicts再合并）
删除新建分支：git branch -d 'branch'

更新操作

替换本地改动：git checkout -- 'filename'（ 使用 HEAD 中的最新内容替换掉工作目录中的文件。已添加到暂存区的改动以及新文件都不会受到影响。）

查看待传文件：git status

添加至暂存区：git add .（ . 修改的+新增的数据；-u=-update 删除的+修改的数据； 'filename' 指定文件名；-A=--all 删除的+替换的+修改的+新增的数据 ）
提交并备注：git commit -m '代码提交信息'
推送至远端：git push origin branch（branch对应分支名）

获取提交ID ：git log
创建版本标签：git tag 'v3.0'

远端拉至本地：git pull = git fetch && merge（获取、合并远端改动）
获取最新版本：git fetch origin

    npm install -g npm@latest或npm install npm -g

hexo命令

升级hexo：npm update hexo -g

新建博客项目

  1. 初始化本地文件夹为博客根目录：hexo init HEXO
    （HEXO为自由命名的文件夹名称，指初始化目录的路径，若无指定则默认为当前目录）
    进入这个文件夹：cd HEXO）
  2. 安装必备的组件：npm install
  3. 安装Hexo：npm install -g hexo-cli

更新文章

  • 新建博客：hexo n "文章标题"（在source_posts生成同名的markdown文件）
    hexo n draft 初稿（在source_drafts生成 初稿.md 文件）
    （draft指定文章类型，若无则默认为default_layout配置）
    （文章标题必填，若含空格则用英文双引号包围）

  • 查看已有草稿：hexo --draft

  • 编辑完草稿后把文章推送到_posts目录下：hexo publish 初稿.md

生成网页

  • 清除缓存：hexo clean（网页显示异常时执行）
  • 生成静态文件：hexo g = hexo generate
  • 启动本地预览：hexo s = hexo server
    → http://localhost:4000/（Hexo会监视文件变动并自动更新）
    默认4000，换用其它端口：hexo s -p 8080
    默认0.0.0.0，指定IP地址：hexo s -i 0.0.0.1
    静态模式 ，仅预览 \public\ 的文件并禁用文件监视：hexo s -s（修改md文件或css文件后不会同时更新本地预览）
    调试模式，将消息详细记录到终端和 debug.log 文件：hexo --debug
    安全模式，禁用加载插件和脚本：hexo --safe（安装新插件遇到问题时执行）
    静默模式，静默输出到终端：hexo --silent
    关闭本地预览：Ctrl＋C

发布网页

部署到github：hexo d = hexo deploy（配置好 _config.yml 的 deploy 再部署）
生成静态文件后部署：hexo g -d = hexo d -g
部署：hexo clean && deploy
